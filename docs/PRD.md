# 微聊 AI 助手 — 产品需求文档 (PRD)

> 版本: v1.1 | 更新: 2026-04-06
> 状态: 开发中，核心功能已上线

## 一、产品概述

**产品名称：** 微聊 AI 助手
**一句话描述：** 让 AI 住进你的微信，聊天省钱两不误
**核心定位：** 以 AI 聊天为钩子，CPS 返利为盈利引擎的微信 Bot SaaS 平台
**差异化：** BotFox 没有 CPS 插件，我们的 CPS 返利 + 积分裂变是核心壁垒

### 商业模型（赚钱路径）

用户注册 → 10000 积分（约3天免费）
AI 聊天（消耗积分）→ 每条 -10 积分
积分用完 → 提示"邀请好友送 5000 积分"
邀请人 +5000，被邀请人 +3000（双向激励，推动裂变）

同时：所有用户发「外卖/淘口令」→ CPS 返利（永久免积分）
用户下单 → 平台佣金（外卖单佣金 2-8 元，电商单佣金 1-30%）

### 盈利测算（保底估算）

| 指标 | 假设 | 依据 |
|------|------|------|
| 日活用户 | 100 人 | 活跃运营 1 个月 |
| CPS 触发率 | 60% 的用户每天至少触发 1 次 | 外卖是高频场景 |
| CPS 转化率 | 20% | 发了链接点开 + 下单的比例 |
| 日均 CPS 订单 | 12 单 | 100 x 60% x 20% = 12 |
| 平均佣金 | 3 元/单 | 外卖 2-8 元取中值 |
| 日收入 | 36 元/天 | 12 x 3 |
| 月收入 | 约1000 元/月 | 36 x 30 x 0.9 |
| 服务器成本 | 约100 元/月 | 腾讯云轻量（已有） |

结论：100 个日活用户就能覆盖成本并有盈余。规模做大后（1000+ 日活），月入可达万元级别。

## 二、目标用户

### 主要画像
- 对 AI 感兴趣但不会装软件的普通微信用户
- 有外卖/网购习惯、对省钱敏感的年轻人
- 愿意分享好东西给朋友的社交活跃用户

### 获客渠道
- 桔梗公司现有业务群（万单联盟用户）→ 直接导入
- 微信公众号/朋友圈分享推广链接（邀请码裂变）
- 小红书/抖音种草笔记

### 使用场景
1. 用户注册 → 创建 Bot → 微信扫码绑定 → Bot 自动回复消息
2. 粉丝发「外卖」→ 返回美团/饿了么红包链接（万单联盟 CPS）
3. 粉丝发淘口令/京东链接 → 返回优惠券链接（订单侠 CPS）
4. 粉丝正常聊天 → AI 回复并自然引导省钱功能
5. 粉丝发「查券 连衣裙」→ 搜索优惠券商品列表

## 四、功能模块

### 4.1 用户注册 & 微信绑定

- 手机号 + 密码注册（无短信验证码）
- 注册送 10000 积分（约够 3-5 天 AI 聊天）
- 邀请码：邀请人 +5000 积分，被邀请人 +3000 积分
- 微信扫码绑定流程：调 iLink API 获取二维码 → 用户扫码 → 保存 bot_token → 启动消息轮询

### 4.2 AI 聊天（默认插件，fallback 模式）

- 用户的微信好友发消息 → AI 自动回复
- 接口：OpenAI 兼容 /chat/completions（当前 API Key 未配置，返回 fallback 文案）
- 上下文：每用户最近 10 轮对话（内存 Map）
- 消耗积分：每条回复扣 10 积分
- 积分用完 → AI 聊天暂停，但 CPS 返利继续工作

### 4.3 CPS 返利插件（默认开启，不可关闭，永久免积分）

> 详见 ARCHITECTURE.md 的 CPS 链路章节

#### 4.3.1 万单联盟（外卖/本地生活）
- 关键词触发：外卖/美团/闪购/酒店/饿了么
- 返回对应推广活动链接（带推广位 ID）
- API：GET /activities，HMAC-SHA256 签名认证，10 分钟缓存

#### 4.3.2 订单侠（电商聚合）
- 淘口令转链：正则匹配 → wn_convert (点卡) / tc_general_convert (套餐) 接口
- 京东转链：链接含 jd.com → by_unionid_promotion 接口
- 拼多多转链：链接含 pinduoduo.com → pdd/convert 接口
- 搜索优惠券：用户发「查券 xxx」→ super_search_material 接口

#### 4.3.3 定时推送
- 每天 10:30 / 16:30 推送外卖红包
- 用户可在控制台开关推送（push_enabled 字段已预留）
- [待实现]

### 4.4 积分系统

| 行为 | 积分 |
|------|------|
| 注册 | +10000 |
| 填邀请码（邀请人） | +5000 |
| 填邀请码（被邀请人） | +3000 |
| 签到 | +?（待实现） |
| AI 聊天（每条） | -10 |
| CPS 消息 | 免费 |

### 4.5 插件系统（22 个内置插件）

| 分类 | 插件 |
|------|------|
| 游戏 | 大富翁、文字 RPG、小农场、侦探解谜、漂流瓶 |
| 工具 | 天气、翻译、去水印、热搜、汇率、音乐VIP解析、音乐解析、文档下载、网盘解析、表情制作、AI绘图、电影搜索、小红书文案 |
| AI | AI 聊天 |
| CPS | CPS 返利（不可关闭） |
| 飞书 | 文档、表格、审批、知识库、消息、通讯录 |

- 内置插件：is_builtin=1，代码在 src/plugins/builtin/
- 社区插件：JS 代码通过沙箱执行（runPluginCode）
- 创建 Bot 时自动安装所有内置插件

### 4.6 Web 控制台

7 个页面（微信绿色主题 #07C160，纯 HTML + Tailwind 零构建）：
- 首页 — 产品介绍
- 登录/注册 — 手机号 + 密码
- 控制台 — Bot 状态、消息记录、积分余额、插件管理
- 微信绑定 — 扫码绑定
- 插件市场 — 插件安装/配置
- 管理后台 — 管理员运营（用户/Bot/插件/消息统计）

## 七、风险矩阵

| 风险 | 概率 | 影响 | 对策 |
|------|------|------|------|
| 微信封号/限制 Bot | 中 | 高 | iLink 是官方接口，风险低于灰色协议；准备备用手机号 |
| CPS 平台调佣金/关停 | 低 | 中 | 同时接入万单联盟 + 订单侠双渠道，不依赖单一平台 |
| AI 成本失控 | 中 | 中 | 积分制兜底，积分用完自动停止 AI 回复；设每日消耗上限 |
| 竞品跟进 CPS 功能 | 低 | 低 | BotFox 不做 CPS 是有原因的（合规风险），我们主动做 |
| 服务器宕机 | 低 | 高 | PM2 自动重启；考虑加监控告警 |
| iLink API 变更/下线 | 低 | 高 | 直连 HTTP API 比 SDK 更容易适配；预留 Telegram 适配 |

## 九、决策记录

| 决策 | 原因 |
|------|------|
| 不用好单库 | 需招商淘客资质，普通开发者无法申请 |
| 不用折淘客 | 官网打不开（可能已跑路） |
| 选订单侠 | 用自己的 PID，不抽佣，只收接口服务费 |
| 去掉 SDK | 直接调 iLink HTTP API，减少依赖层 |
| SQLite | 单机部署够用，运维简单 |
| 纯 HTML | 零构建，CDN 被墙也不影响 |
| CPS 不可关闭 | 产品决策 — CPS 是核心盈利通道 |
| CPS priority=10 | 确保 CPS 消息最先处理，不被其他插件抢走 |
